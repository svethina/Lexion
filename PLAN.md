# План: кнопки AI («О чём статья», «Тезисы», «Пост для Telegram»)

На основании [PROJECT.md](./PROJECT.md).

## Текущее состояние

| Функция | Статус |
|---------|--------|
| Парсинг статьи по URL | ✅ `POST /api/parse` |
| Перевод статьи | ✅ `POST /api/translate` |
| «О чём статья» | ⏳ заглушка в UI |
| «Тезисы» | ⏳ заглушка в UI |
| «Пост для Telegram» | ⏳ заглушка в UI |

Общая схема для всех трёх кнопок:

```
URL → парсинг (cheerio) → текст статьи → OpenRouter (DeepSeek) → результат в блоке «Результат»
```

---

## Шаг 1. Вынести общую логику AI-действий

**Файл:** `lib/article-ai.ts`

1. Описать тип действия: `summary` | `theses` | `telegram-post`.
2. Сделать функцию подготовки текста статьи (title, date, content) — переиспользовать подход из `translateArticle`.
3. Для каждого действия задать **system prompt** и **user prompt**:
   - **summary** — кратко на русском: о чём статья, 3–5 предложений.
   - **theses** — тезисы на русском: маркированный список, 5–10 пунктов.
   - **telegram-post** — пост для Telegram на русском: цепляющий заголовок, 2–4 абзаца, уместные эмодзи, до ~1500 символов.
4. Экспортировать `generateArticleAction(action, article)`.

**Зависимости:** `lib/openrouter.ts` (`chatCompletion`), `lib/parse-article.ts` (`ParsedArticle`).

---

## Шаг 2. API-эндпоинт для действий AI

**Файл:** `app/api/ai/route.ts`

1. Принимать `POST` с телом: `{ url: string, action: "summary" | "theses" | "telegram-post" }`.
2. Валидировать URL (как в `/api/parse` и `/api/translate`).
3. Вызвать `parseArticle(url)`.
4. Если `content` пустой — вернуть `422` с понятной ошибкой.
5. Вызвать `generateArticleAction(action, article)`.
6. Вернуть JSON: `{ result: string }`.

Опционально: один эндпоинт на действие (`/api/summary`, `/api/theses`, `/api/telegram-post`) — если предпочтительнее явное разделение.

## Шаг 3. Подключить кнопки в интерфейсе

**Файл:** `app/components/ArticleForm.tsx`

1. Переименовать кнопку «Пост» → **«Пост для Telegram»** (по требованию).
2. Заменить заглушку в `handleAction` на `fetch("/api/ai", ...)`.
3. Передавать `url` и `action` в теле запроса.
4. Показывать `data.result` в блоке «Результат» (обычный текст, не JSON).
5. Обновить тексты загрузки: «Генерация «О чём статья»…» и т.д.
6. Обрабатывать ошибки API (403 парсинга, нет ключа OpenRouter, таймаут).

---

## Шаг 4. Промпты и качество ответов

**Файл:** `lib/article-ai.ts` (или `lib/prompts.ts`)

1. Зафиксировать промпты в константах для удобной правки.
2. Указать в system prompt: язык ответа — **русский**, без вводных фраз вроде «Вот тезисы».
3. Для **telegram-post** явно описать формат:
   - заголовок;
   - основной текст;
   - всегда указывать ссылка на источник (URL статьи);
   - лимит символов под Telegram.
4. Прогнать 2–3 тестовых URL и подкрутить промпты при необходимости.

---

## Шаг 5. Переменные окружения и деплой

1. Убедиться, что на Vercel заданы:
   - `OPENROUTER_API_KEY`
   - `OPENAI_BASE_URL`
2. После изменений — **Redeploy** на Vercel.
3. Локально — ключ в `.env.local`, перезапуск `pnpm dev`.

---

## Шаг 6. Проверка

| Сценарий | Ожидание |
|----------|----------|
| Валидный URL + «О чём статья» | Краткое описание на русском |
| Валидный URL + «Тезисы» | Список тезисов на русском |
| Валидный URL + «Пост для Telegram» | Готовый пост для публикации |
| URL без текста | Ошибка «текст статьи не найден» |
| Сайт с HTTP 403 | Понятное сообщение о блокировке |
| Нет `OPENROUTER_API_KEY` | Сообщение про `.env.local` / Vercel |
| `pnpm build` | Сборка без ошибок |
